# Screenshots

Store review images for user-visible changes here only when they add durable value beyond a pull-request attachment.

Use lowercase descriptive filenames in the form `yyyy-mm-dd-feature-state.png`, for example `2026-07-18-home-empty.png`. Remove obsolete images when the documented interface changes.

Pull requests with visual changes should include before/after images or a short recording in the pull-request description. Do not commit screenshots containing personal paths, user Workspace data, account names, or other private information.
